Persistencia de Información, Roles de Acceso Seguro y Gestión Gráfica

6.1 Creación de la Base de Datos en el Servidor

La persistencia de los datos en aplicaciones web dinámicas depende de un motor de bases de datos relacionales, siendo **MySQL** el estándar global en servidores web. El proceso de inicialización requiere declarar el espacio lógico de almacenamiento y configurar el cotejamiento correcto de caracteres (habitualmente utf8mb4_general_ci) para dar soporte nativo a caracteres especiales e idiomas internacionales.

En entornos cPanel, esta tarea se ejecuta mediante el asistente "Bases de datos MySQL", el cual añade automáticamente un prefijo del nombre de usuario de la cuenta para garantizar el aislamiento físico entre diferentes cuentas compartidas en la misma infraestructura.

6.2 Creación de Usuarios y Asignación de Privilegios

Por directrices internacionales de seguridad informática, las aplicaciones web **nunca** deben conectarse utilizando el usuario administrador global (root). En su lugar, se implementa el principio de menor privilegio, consistente en:

1. Creación de Entidad

Declaración de un usuario exclusivo con contraseña robusta cifrada para restringir su uso únicamente a los scripts internos del servidor.

2. Vinculación y Permisos

Asignación estricta de privilegios operativos (SELECT, INSERT, UPDATE, DELETE) limitados exclusivamente a la base de datos del proyecto.

💻 Referencia de Sentencias SQL de Configuración

CREATE DATABASE db_proyecto; -- Crea el esquema lógico
CREATE USER 'user_web'@'localhost' IDENTIFIED BY 'K3#9pL$zM!';
GRANT SELECT, INSERT, UPDATE, DELETE ON db_proyecto.* TO 'user_web'@'localhost';
FLUSH PRIVILEGES; -- Aplica los cambios inmediatamente

6.3 Administración de Datos mediante phpMyAdmin

**phpMyAdmin** es la herramienta gráfica web por excelencia programada en PHP para la gestión directa de motores MySQL. Proporciona una interfaz visual robusta que abstrae la complejidad sintáctica, permitiendo realizar con clics las siguientes tareas críticas de administración:

  • Estructuración: Diseño visual de tablas, definición de llaves primarias, foráneas e índices.
  • Mantenimiento: Importación y exportación de respaldos en archivos de texto plano `.sql`.
  • Operaciones SQL: Ejecución de consultas personalizadas, optimización de índices y reparación de tablas dañadas.
