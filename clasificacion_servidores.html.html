Módulo II: Clasificación de los Servidores

Infraestructura, Virtualización y Control por Consola

2.1 Características Principales de un Servidor

Un servidor es una computadora de altas prestaciones optimizada para proveer recursos, datos, servicios o programas a otras computadoras conocidas como clientes a través de una red. Sus características fundamentales son:

  • Alta Disponibilidad: Operación continua (24/7/365) mediante hardware tolerante a fallos.
  • Escalabilidad: Aptitud para incrementar sus capacidades físicas (RAM, CPU) sin perder rendimiento.
  • Procesamiento Concurrente: Habilidad para responder simultáneamente miles de solicitudes de clientes de forma aislada y segura.

2.2 Información Principal del Equipo (Software y Hardware)

Procesamiento (Hardware)

Uso de CPUs multinúcleo empresariales (Intel Xeon / AMD EPYC) que soportan altas cargas operativas y memoria ECC RAM.

Almacenamiento Concurrente

Arreglos de discos NVMe o SAS configurados en RAID redundantes que garantizan la integridad de los datos ante fallos físicos.

Kernel Corporativo (Software)

Sistemas robustos desprovistos de interfaces pesadas para priorizar hilos de ejecución dedicados (Ubuntu Server / Windows Server).

2.4 Instalación de Linux y de Windows

Fase 01

Montaje de la Imagen ISO

Se indexa la imagen descargada del sistema operativo (Ubuntu .iso o Windows Server .iso) en el hipervisor.

Fase 02

Configuración del Idioma y Particionado

En Linux se prefiere un particionado ext4 sobre LVM. En Windows se formatean los volumenes bajo NTFS primario.

2.5 Primeros Pasos en el Manejo de Linux y de Windows

Una vez completada la instalación, la fase de inicialización técnica y post-instalación requiere asegurar los canales de comunicación y actualizar los repositorios de software centrales:

En Linux Server

El primer inicio es puramente en modo texto. La acción obligatoria consiste en actualizar los índices de paquetes con los servidores espejo globales e iniciar el demonio de SSH para la administración remota segura.

En Windows Server

El sistema despliega el cuadro de control "Administrador del Servidor". Los primeros pasos implican asignar una dirección IP estática fija a la tarjeta de red interna, cambiar el nombre genérico del equipo y habilitar el Escritorio Remoto (RDP).

2.6 Comandos Básicos de Consola (CLI)

La administración interactiva multimedia de los servidores se realiza mediante interfaces de línea de comandos (Bash en Linux y PowerShell / CMD en Windows). Aquí se detallan los comandos de control esenciales:

💻 Consola de Referencia Linux (Bash Terminal)

root@servidor:~# pwd
/root (Muestra el directorio de trabajo actual)
root@servidor:~# ls -lh
drwxr-xr-x 2 root root 4.0K May 17 16:30 public_html (Lista archivos con permisos)
root@servidor:~# sudo apt update && sudo apt upgrade -y
Leyendo lista de paquetes... Hecho. (Actualiza el software del servidor)
root@servidor:~# ifconfig
inet 192.168.1.100 netmask 255.255.255.0 (Muestra la dirección IP del nodo)

💻 Consola de Referencia Windows (PowerShell)

PS C:\Users\Administrator> Get-Location
Path: C:\Users\Administrator (Equivalente a pwd en Linux)
PS C:\Users\Administrator> Get-ChildItem
Directory: C:\Users\Administrator (Muestra el árbol de carpetas)
PS C:\Users\Administrator> Test-Connection -ComputerName 8.8.8.8
Source Destination IPStatus Time(ms)
Server 8.8.8.8 Success 12 (Verifica conectividad externa de red)
