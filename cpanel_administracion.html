Interfaces de Gestión, Control de Archivos y Protocolos de Transferencia

4.1 cPanel de Servidor Web

El **cPanel** es un panel de control gráfico simplificado basado en Linux que automatiza la administración de servicios de alojamiento web. Funciona mediante un cuadro de mando de administración integral que permite delegar tareas complejas de gestión de sistemas a través de una interfaz interactiva de clics intuitivos. Sus facultades críticas incluyen:

  • Gestión de Dominios: Creación y mapeo de subdominios, alias de correo y redireccionamientos DNS.
  • Administración de Datos: Creación de bases de datos relacionales MySQL / MariaDB y control de accesos vía phpMyAdmin.
  • Seguridad Integrada: Instalación de módulos SSH/TLS, restricción de direcciones IP y firewalls de aplicaciones web.

4.2 Administrador de Archivos

Es la herramienta web nativa del panel que permite manipular directamente la estructura del árbol de directorios del servidor remoto. Evita depender obligatoriamente de software externo de terceros al permitir cargar, comprimir, extraer, renombrar y editar archivos de código fuente en tiempo real.

📍 Espacio de Despliegue Crítico:
El núcleo central de la visibilidad en Internet reside en el directorio público public_html. Este es el espacio exclusivo donde deben alojarse los documentos raíz del proyecto (como el archivo index.html) para que el software del servidor web los reconozca y despache de manera pública a las solicitudes del navegador.

4.3 Protocolo de Transferencia de Archivos (FTP)

El protocolo **FTP** (File Transfer Protocol) es el estándar de red utilizado para transferir archivos masivos de datos entre un sistema cliente local y el servidor remoto. Utiliza un modelo de arquitectura de conexión de control por el puerto **TCP 21** y transferencia de flujo de datos por el puerto **TCP 20**. Para operaciones de desarrollo profesional se emplean clientes avanzados de escritorio como *FileZilla* bajo conexiones cifradas seguras (**SFTP**).
